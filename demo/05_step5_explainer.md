# Demo 第 5 步：Reporter Agent

> 对应 `05_reporter.py`，TSci 论文里的最后一个 Agent。
> 读完你会理解：(1) 为什么 production agent 写报告时要把 LLM 的"自由度"压到最低，(2) 怎么用代码 + LLM 分工把一份报告写得既有人味又有数字可信度。

---

## 1. 这一步做什么？

| 输入 | 处理 | 输出 |
|---|---|---|
| 三份 state JSON + 两张 PNG | 代码摊平事实 → LLM 写散文 → 代码组装 markdown | `final_report.md` |

---

## 2. 核心设计：LLM 只写散文，代码负责所有数字

为什么不让 LLM 一把把整份报告写出来？

**两个反复出现的失败模式**：

1. **数字漂移**：哪怕 prompt 里给了 `val_mape=2.573`，LLM 写报告时可能写成 `约 2.6` 或更糟，写成 `2.857`。十次有一两次会编。
2. **结构漂移**：每次跑出的报告章节顺序、表格列名、字段命名都不一样，下游解析（grep / 自动化测试）会炸。

**我们的分工**：

| 谁 | 负责 |
|---|---|
| **代码** | 表格、数字、图片嵌入、章节顺序、所有 markdown 结构 |
| **LLM** | 5 段散文：摘要 / 数据洞察 / 选型解释 / 集成解释 / 局限性 |

LLM 输出的 JSON 就是 5 个键值对，代码把它们填到固定模板的固定槽位。

> 这跟前几步"LLM 输出严格 JSON"是同一个思想的延伸——**让 LLM 的输出空间尽可能小、可校验**。

---

## 3. 散文的 5 个槽位为什么是这 5 个？

参考 TSci 论文的报告评估维度（分析严谨性 / 模型论证 / 解释连贯性 / 可操作性 / 结构清晰度），我们把对应的"需要主观表达"的部分抽成 5 段：

| 字段 | 内容 | 字数 |
|---|---|---|
| `executive_summary` | 总览：数据、方法、精度 | 80-180 |
| `data_insight` | 引用 Q/A 的具体数字描述数据特征 | 80-180 |
| `model_selection_rationale` | 为何选这些候选 + 引用 val_MAPE | 80-180 |
| `ensemble_decision_rationale` | 为何选这种集成策略 + 引用权重 | 80-180 |
| `limitations` | 诚实写出风险（这是研究价值所在） | 120-200 |

**重点**：每段 prompt 里都写明"必须引用具体数字"，把 LLM 的注意力锚在 facts 上。

---

## 4. limitations 字段为什么单独强调？

这是整份报告**最有研究价值**的一段。一个会"说人话"的 LLM 容易被表面的好结果（test_MAPE=2.16%）带偏，写出过分乐观的结论。

我们 prompt 里列出了 4 个必须诚实写出的风险：

1. **训练数据是合成的** —— 真实序列复杂得多
2. **未做分布偏移检验** —— test 集只有 14 天，模型上线后可能马上失效
3. **Curator 诊断没有置信度** —— A 字段是点估计，小样本时会撒谎（论文末尾的核心局限）
4. **模型库手工固定** —— 遇到库外模式就废

> 这正是 TSci 论文留下的研究入口。Reporter 主动写出局限性 = 把"下次怎么改进"的钩子留给读者。

---

## 5. 代码侧的表格生成

报告里 4 张表（Q 指标、A 维度、候选 + 理由、test 结果、超参试验）都是 `fmt_xxx_table()` 一次拼好。每张表都遵守：

- **列名固定**：方便后续脚本/网页自动解析；
- **数字格式化**：`f"{x:.3f}"` 统一保留 3 位小数；
- **嵌入图片用相对路径**：`![](curator_panel.png)`，假定报告和图片在同目录——这样无论是 VSCode / GitHub 还是本地预览器都能正确显示。

---

## 6. 报告结构

```
# 时序预测项目报告
## 执行摘要              ← LLM
## 1. 数据画像           ← LLM 散文 + 代码 Q/A 表 + 图
## 2. 模型选型           ← LLM 散文 + 代码候选表
## 3. 集成决策           ← LLM 散文 + 代码权重列表
## 4. 预测结果           ← 代码 test 表 + 图
## 5. 假设与局限性       ← LLM
## 附录 A 全部超参试验   ← 代码
## 附录 B 任务设置       ← 代码
```

**附录的意义**：让任何审查者都能复现你的决策路径——所有 trial 都列在表里，没有藏。这是 production-grade 报告的最低标准。
